# GeoLogic - Demo Walkthrough

This guide walks you through the features of GeoLogic, including the new AI capabilities (Milestone 3).

## Prerequisites
Ensure the stack is running:
- **Frontend**: `http://localhost:3000`
- **Backend**: `http://localhost:8000`
- **Database**: PostgreSQL (Docker)

## Scenario 1: User Location Validation (AI Explained)
1.  Open **[http://localhost:3000](http://localhost:3000)** in your browser.
2.  Click **"Validate Location"**.
3.  Observe the result:
    -   **Decision**: `ALLOW`
    -   **Explanation**: notice the explanation text is generated by OpenAI.

## Scenario 2: AI Policy Generation
1.  Navigate to **[http://localhost:3000/admin/policies](http://localhost:3000/admin/policies)**.
2.  Locate the "âœ¨ Generate Policy with AI" box at the top.
3.  Type a natural language prompt, for example:
    > "Allow US, Canada, and Mexico. strict block on VPNs. require accuracy better than 50 meters."
4.  Click **"Generate"**.
5.  Watch the JSON Editor update automatically:
    -   `allowed_countries` should include `["US", "CA", "MX"]`.
    -   `vpn_handling.mode` should be `"DENY"`.
    -   `gps_rules.max_accuracy_m` should be `50`.
6.  Click **"Save & Activate Policy"** to apply this new rule.

## Scenario 3: Admin Audit Trail
1.  Navigate to **[http://localhost:3000/admin/audit](http://localhost:3000/admin/audit)**.
2.  Verify your latest attestations are logged.

## AI Configuration
OpenAI is required for policy generation and decision explanations.
1.  Add `OPENAI_API_KEY=sk-...` to `apps/api/.env`.
2.  Restart the backend.
